# WebP Lossless Images
WebP is a new type of image format that can provide high-quality lossless and lossy compression for images on the Internet. With WebP, website administrators and web developers can create smaller but more detailed images, thereby enhancing web access speed.

Download and install WebP
[webp](https://developers.google.cn/speed/webp/download?hl=zh-cn)
### Windows
After downloading and decompressing, you can directly execute the command in the bin directory. No further elaboration is needed!

### Linux
Install the libjpeg, libpng, libtiff and libgif packages to convert between JPEG, PNG, TIFF, GIF and WebP image formats.

Package management varies by Linux distribution. On Ubuntu and Debian, the following command will install the required software packages:
` ` `
yum -y install libjpeg-dev libpng-dev libtiff-dev libgif-dev
` ` `
Unzip the downloaded tar xvzf libwebp-1.5.0.tar.gz
` ` `
cd libwebp-1.5.0
./configure
make
sudo make install
` ` `
This will build and install the cwebp and dwebp command-line tools as well as the libwebp library (dynamic and static).

By default, these tools are usually installed under /usr/local/bin/. The local version is built in the examples/ directory.

This library is usually installed in the /usr/local/lib/directory. To avoid runtime errors, please ensure that your LD_LIBRARY_PATH environment variable contains this location. C header files are usually installed under /usr/local/include/webp.

Use
Use cwebp to convert images to WebP format
Use cwebp in the command line to convert PNG or JPEG image files to WebP format. You can convert PNG image files to WebP images with quality ranges (a total of 80) :

` ` `
cwebp [options] input_file -o output_file.webp
` ` `
Common Options

- ***-o string***
Specify the name of the output WebP file. If omitted, cwebp will perform compression but only report statistical information. Using "-" as the output name will direct the output to "stdout".

- ***-q float***
Specify the compression coefficient of the RGB channel, ranging from 0 to 100. The default value is 75.
If it is lossy compression (default), a smaller coefficient will result in a lower-quality file. Using a value of 100 can achieve the best image quality.

- ***-z int***
Enable the lossless compression mode and specify the level (between 0 and 9), where level 0 indicates the fastest and level 9 indicates the slowest. Compared with the slower mode, the fast mode generates larger file sizes. It is recommended to use -z 6 as the default value. This option is actually a shortcut to some predefined Settings for quality and methods. If the options -q or -m are used subsequently, they will invalidate this option.

- ***-mt***
If possible, please use multi-threading for encoding.

Other options to see website [cwebp] (https://developers.google.cn/speed/webp/docs/cwebp?hl=zh-cn)
